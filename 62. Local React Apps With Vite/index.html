Local React Apps With Vite - Core Concepts & Key Lessons

1. Why Run React Locally?

CodeSandbox is useful for learning React quickly, but running React locally feels more realistic and removes dependency on internet access or online tools.

  • Local development feels closer to real-world workflows
  • Code is identical to CodeSandbox
  • You control the environment

2. What Is Vite?

Vite is a modern tool for quickly creating and running React applications locally. It is pronounced “veet”.

  • Extremely fast setup
  • Includes a development server
  • Handles JSX transpilation
  • Supports React, Vue, Svelte, and more

Vite does all the heavy lifting so you can focus on writing React code.

3. Creating a React App With Vite

To create a new React app, run the following command in an empty folder: 


npm create vite@latest

During setup:

  • Choose a project name (folder name)
  • Select React as the framework
  • Select JavaScript (not TypeScript)

4. Installing Dependencies & Running the App

After creating the project:


cd project-name
npm install
npm run dev

This starts a local development server with live reloading. Any file changes automatically refresh the browser.

5. Create React App (CRA) vs Vite

Create React App was the old standard way to create React apps.

  • Still works
  • Slower startup
  • Older tooling

Vite is now preferred:

  • Faster
  • More flexible
  • Recommended in modern React workflows

6. Vite Project Structure

Important folders and files:

  • index.html – base HTML file
  • src/ – where all React code lives
  • src/main.jsx – renders the App component
  • src/App.jsx – root React component
  • src/index.css – global styles

7. Rendering the App

React renders the App component into a <div id="root"> inside index.html.

The file main.jsx handles this process. You usually do not modify it.

8. Creating Components in Vite

Components live inside the src folder.

  • Use .jsx or .js extensions
  • Component names are capitalized
  • Components are exported and imported normally

React code written in Vite is identical to CodeSandbox.

9. Styling Components

Styling in Vite follows the same conventions used earlier:

  • Component-specific CSS files (e.g. Chicken.css)
  • Import CSS into the component file
  • Use className instead of class

CSS is included in the final build only if it is imported.

Key Takeaways

  • Vite is the modern way to create React apps locally
  • Setup is fast and simple
  • Live reloading works automatically
  • React code is identical to CodeSandbox
  • Vite handles Babel, bundling, and dev server setup
