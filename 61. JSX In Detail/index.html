JSX in Detail – Core Concepts & Key Lessons

1. Importing & Exporting Components

React applications are split into many files. Typically, each component lives in its own file.

  • One component per file (recommended)
  • File name should match component name
  • Most components use default exports

Components must be exported before they can be imported. Files do not automatically know about each other.

2. Core Rules of JSX

Rule 1: Self-closing tags must close

JSX requires explicit closing for self-closing elements.


<input />
<img />
<br />

Rule 2: Only one top-level element may be returned

Components can return many elements, but they must be wrapped in a single parent element. 


<div>
  <h1>Hello</h1>
  <p>Welcome</p>
</div>

Parentheses are often used to format JSX cleanly across multiple lines.

3. React Fragments

React Fragments allow you to satisfy JSX’s single-root rule without adding unnecessary HTML elements to the DOM. 


<>
  <input />
  <input />
  <button>Login</button>
</>

Fragments render no extra HTML. Only their children appear in the DOM.

4. JavaScript Expressions in JSX

Curly braces {} allow JavaScript expressions inside JSX.

  • Variables
  • Math expressions
  • Function calls
  • Object properties

<h2>{petName}</h2>
<p>{price * nights}</p>

Only expressions are allowed. Not statements like if or for.

5. JSX in Real Components (Die Example)

A React component is just a JavaScript function. It can run logic and return JSX using computed values.

  • Variables are created inside the function
  • Values are injected into JSX with {}
  • Each render runs the function again

6. Component Decomposition

React encourages breaking UIs into small, reusable components and assembling them into larger ones. 


App
 └── DiceRoll
      ├── Die
      ├── Die
      └── Die

This keeps code reusable, readable, and scalable.

7. Styling JSX Components

class → className

Because class is a reserved JavaScript keyword, JSX uses className. 


<h2 className="die">4</h2>

CSS Files

  • CSS is written normally
  • CSS files are imported into components
  • Importing CSS tells Webpack to include it

import "./Die.css";

CSS is still global. File separation is a convention, not scoping.

Key Takeaways

  • JSX is strict but predictable
  • Components must return one root element
  • Fragments avoid unnecessary DOM nodes
  • {} makes JSX dynamic
  • Components should be small and composable
  • Styling uses className and imported CSS
