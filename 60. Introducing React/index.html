Introduction React: Core Concepts & Key Lessons

1. What React Is

React is a front-end JavaScript library used to build user interfaces. Its main purpose is to help developers create complex UIs by breaking them into small, reusable pieces called components.

Instead of building entire pages at once, React encourages thinking in terms of individual UI elements (buttons, cards, forms, lists) that can be reused and combined.

2. Components

Components are the core building blocks of React applications. A component is usually a JavaScript function that knows how to render itself to the screen.

  • Components combine logic (JavaScript) and presentation (HTML/CSS)
  • Components can be reused multiple times
  • Components can contain other components

Large applications are built by nesting many small components into larger ones.

3. JSX

JSX stands for JavaScript Syntax Extension. It allows developers to write HTML-like code directly inside JavaScript files.

Although JSX looks like HTML, it is not real HTML and is not valid JavaScript on its own.

Example conceptually:


function App() {
  return <h1>Hello World</h1>;
}

4. Babel

Babel is a tool that converts JSX into real JavaScript that browsers can understand.

  • JSX is written by the developer
  • Babel transpiles JSX into JavaScript function calls
  • The browser ultimately receives standard JavaScript

This process happens automatically in modern React setups, so developers do not need to manually use Babel.

5. Basic React App Structure

Most React applications follow a standard structure. There is a top-level component called App.

  • App is the root component of the application
  • All other components are rendered inside App
  • index.js renders App into a DOM element

Typically, React renders the App component inside a <div id="root"> in the HTML file.

6. Component Hierarchy

React applications are structured as a hierarchy (or tree) of components.

Example hierarchy:


App
 ├── Navbar
 ├── SearchBar
 └── VideoList
      └── Video
           ├── Thumbnail
           └── LikeButton

Each component has a single responsibility and can be reused across the application.

Key Takeaways

  • React is about building reusable components
  • Components are JavaScript functions that return JSX
  • JSX looks like HTML but is transpiled by Babel
  • The App component is the root of the UI
  • Complex UIs are built by nesting simple components
